# CT Investments 程式碼完整說明

> 本文件以白話文說明「謙堂資本」(CT Investments) 專案中每一支主要檔案的功能與運作方式。適合非工程背景的人閱讀，方便日後維護與理解。

---

## 一、整體架構

### 1.1 這個專案在做什麼？

CT Investments 是一個「台灣股票技術分析儀表板」，讓使用者可以：

- 看到台灣上市、上櫃、興櫃股票的即時行情
- 查看三大法人（外資、投信、自營商）的買賣超排行
- 對個股進行技術分析（K 線圖、RSI、MACD、KD、布林通道等）
- 查看每日晨訊報告（國際行情、市場情緒、操作建議）
- 管理自己的關注清單、投資組合、到價提醒
- 查看 AI 選股排行、當沖排行、產業熱力圖
- 查看管理者推薦的「謙堂觀點」持股

### 1.2 前後端怎麼溝通？

```
使用者的瀏覽器 (前端)
    |
    | HTTP 請求 (例如 GET /api/picks)
    v
Python 伺服器 (後端 server.py)
    |
    | 從外部 API 抓資料 / 從 SQLite 資料庫讀資料
    v
回傳 JSON 給前端 → 前端用 JavaScript 渲染成畫面
```

- **前端**：純 HTML + CSS + JavaScript，沒有用任何框架（React、Vue 等都沒用）
- **後端**：一支 Python 檔案 `server.py`，只用 Python 內建函式庫，不需要額外安裝任何套件
- **資料庫**：SQLite，一個輕量的本機資料庫檔案
- **部署**：放在 Zeabur 雲端平台，自動從 GitHub 的 main 分支部署

### 1.3 資料從哪裡來？

| 資料來源 | 提供什麼 |
|---------|---------|
| TWSE（證交所） | 上市股票行情、法人買賣超、融資融券、本益比 |
| TPEX（櫃買中心） | 上櫃股票行情、法人買賣超 |
| Yahoo Finance | 國際指數、台股即時報價（備援）、歷史K線 |
| Cnyes（鉅亨網） | 新聞、法說會資訊、台指期即時走勢 |
| TAIFEX（期交所） | 台指期即時報價 |
| Fugle API | 即時個股報價（付費 API） |
| FinMind API | 法人買賣超歷史資料、當沖資料、月營收 |
| MIS（證交所即時） | 盤中即時五秒報價 |

## 二、後端 (server.py)

這是整個系統的「大腦」，負責接收前端請求、去外部抓資料、管理會員系統、處理付款等。全檔約 3,430 行。

### 2.1 設定與初始化

**開頭的環境變數與設定**

- `PORT`：伺服器監聽的埠號，預設 8888
- `JWT_SECRET`：用來加密登入 Token 的密鑰
- `DB_PATH`：SQLite 資料庫存放位置（在 Zeabur 上是 `/data/ct_invest.db`）
- `NEWEBPAY_*`：藍新金流（付款系統）的設定
- `FUGLE_API_KEY`：Fugle 即時報價 API 的金鑰
- `FINMIND_TOKEN`：FinMind 資料 API 的金鑰
- `ALLOWED_HOSTS`：允許代理轉發的外部網站清單（防止被當成跳板）
- `PLAN_HIERARCHY` / `PLAN_PRICING` / `PLAN_FEATURES`：訂閱方案的定義（Free / Pro / Pro+）
- `TESTING_MODE`：設為 1 時，所有使用者都享有 Pro+ 全功能（目前啟用中）

**`_tw_now()`**：取得台灣時間（UTC+8），整個系統都用這個函式來確保時區正確。

### 2.2 每日健康檢查

**`_run_daily_healthcheck()`**

每天早上 8 點（台灣時間，只有工作日）自動執行，檢查 10 個項目：

1. **資料庫連線** — 確認 SQLite 可以正常讀寫，統計會員數、觀點數等
2. **TWSE OpenAPI** — 證交所公開 API 是否正常
3. **TWSE 傳統 API** — 法人買賣超 API 是否正常
4. **TPEX** — 櫃買中心 API 是否正常
5. **Yahoo Finance** — 加權指數和台積電 ADR 報價是否正常
6. **Fugle API** — 即時報價 API 是否正常（有設定金鑰才檢查）
7. **FinMind API** — 法人歷史資料 API 是否正常（有設定 Token 才檢查）
8. **Proxy 功能** — 透過自身代理測試外部 API
9. **觀點 API** — 管理者推薦持股 API 是否正常
10. **晨訊預載** — 觸發當天晨訊的背景生成

**`_healthcheck_scheduler()`**：一個永遠跑不停的背景排程，計算距離下一次早上 8 點還有多久，時間到就執行健檢。

### 2.3 代理快取 (Proxy Cache)

**`proxy_cache_get()` / `proxy_cache_set()`**

為了避免頻繁請求外部 API（尤其是 TWSE 會擋 IP），伺服器端會快取代理結果：

- 一般 API 快取 120 秒（2 分鐘）
- 即時報價 (MIS) 快取 8 秒
- 快取超過 200 筆時會自動清除過期資料

### 2.4 個股新聞

**`_fetch_stock_news(code, name)`**

從鉅亨網 API 搜尋特定股票的相關新聞（14 天內），最多回傳 15 則。會用股票代號去比對新聞的「market code」欄位，或是檢查標題是否包含代號/名稱。結果快取 5 分鐘。

### 2.5 重大事件

**`_fetch_market_events()`**

收集市場重大事件，來源包括：

- 鉅亨網搜尋「法說會」相關新聞
- 鉅亨網頭條新聞（篩選包含法說、除息、併購等關鍵字的）
- Yahoo Finance 主要美股的除息日期

**`_fetch_stock_events(code)`**

針對個股抓取事件（法說會 + 除息日），用來在 K 線圖上標記重要事件。

### 2.6 台指期即時資料

**`_futures_record_tick()` / `_futures_poller()`**

- 背景每 30 秒去期交所 API 抓一次台指期報價
- 日盤時段：08:45 ~ 13:45（週一到週五）
- 夜盤時段：15:00 ~ 隔日 05:00
- 累積的報價用來畫台指期的即時走勢圖

### 2.7 謙堂晨訊 (Morning Report)

這是系統最複雜的功能之一，每天自動產生一份完整的盤前分析報告。

**`_mr_generate()`** — 晨訊生成主流程：

1. **`_mr_get_markets()`**：從 Yahoo Finance 抓取 17 項國際指數行情（S&P 500、道瓊、費半、日經、台積電 ADR、原油、黃金、VIX 等）
2. **`_mr_get_twse()`**：抓取加權指數的最新交易日資料（成交量、成交金額、指數）
3. **`_mr_get_inst()`**：抓取三大法人買賣超彙總，以及個股法人明細（最多 100 檔）
4. **`_mr_get_news()`**：從鉅亨網抓取頭條、台股、國際、匯率、總經新聞
5. **`_mr_get_earnings()`**：篩選含 EPS、營收、財報等關鍵字的新聞
6. **`_mr_calc_sentiment()`**：根據美股、費半、台積電 ADR、VIX、殖利率等計算「市場情緒分數」（1~5 顆星）
7. **`_mr_build_viewpoint()`**：自動撰寫操作建議（純文字分析，例如「美股大漲，費半氣勢強勁...」）

生成完的報告存入 SQLite 資料庫和記憶體快取，當天重複請求就直接讀快取。

### 2.8 大盤回測

**`_load_taiex_history()`**：從 Yahoo Finance 下載加權指數 10 年歷史資料，存入資料庫。

**`_run_backtest(db, strategy, params)`**：支援三種回測策略：

- **`lump_sum`（單筆投入）**：在起始日買入一筆，算到結束日的報酬率、年化報酬率、最大回撤
- **`dca`（定期定額）**：每月固定投入，計算最終價值和報酬率
- **`ma_cross`（均線交叉）**：短均線穿越長均線買入、跌破賣出，計算勝率和報酬率

### 2.9 除權息行事曆

**`_fetch_dividend_raw()`**：從 TWSE 抓取未來 3 個月的除權息資料。

**`_get_dividend_data()`**：可依關注清單或月份過濾。免費版只能看關注清單裡的股票，Pro+ 可以看全市場。

### 2.10 法人連續買賣超統計

**`_collect_inst_daily()`**：每天將法人個股買賣超資料存入 `inst_daily` 資料表。

**`_compute_streaks()`**：計算某檔股票被某類法人連續買超或賣超幾天、累計金額多少。

**`_streak_top()`**：列出連續買超或賣超天數最長的前 20 名股票。

**`_backfill_inst_daily()`**：如果資料庫裡不到 10 天的歷史資料，就從 FinMind API 回填過去 30 個交易日。

### 2.11 Fugle 和 FinMind API

**Fugle**：即時個股報價，有每分鐘 50 次的速率限制。

- `_fugle_quote(code)`：查詢單一股票報價
- `_fugle_batch(codes)`：批次查詢多檔報價（最多 30 檔）

**FinMind**：歷史資料查詢（法人買賣超、當沖統計等），快取 10 分鐘。

### 2.12 資料庫

**`init_db()`** — 建立所有需要的資料表：

| 資料表 | 用途 |
|-------|------|
| `users` | 會員帳號（email、密碼、暱稱、角色、方案） |
| `watchlists` | 每個使用者的關注清單 |
| `user_actions` | 使用者行為紀錄（登入、看了什麼頁面等） |
| `morning_report` | 每日晨訊報告的快取 |
| `stock_picks` | 管理者推薦的觀點持股 |
| `inst_daily` | 每日法人個股買賣超資料 |
| `taiex_history` | 加權指數歷史 K 線（回測用） |
| `portfolios` | 使用者的投資組合 |
| `price_alerts` | 到價提醒設定 |
| `plan_changes` | 方案變更紀錄 |
| `orders` | 付款訂單紀錄 |

如果資料庫是全新的（沒有任何會員），會自動建立一個管理員帳號（chan010212@gmail.com）和預設的觀點持股。

### 2.13 會員系統 (JWT)

**`jwt_encode()` / `jwt_decode()`**：自製的 JWT（JSON Web Token）實作，用 HMAC-SHA256 簽章。Token 有效期 30 天。

**`hash_password()` / `verify_password()`**：密碼用 PBKDF2 演算法加鹽雜湊，不會以明文儲存。

### 2.14 藍新金流付款

**`newebpay_encrypt()` / `newebpay_decrypt()`**：用 openssl 指令做 AES-256-CBC 加解密（因為不裝 pip 套件，所以用 subprocess 呼叫 openssl）。

**`handle_checkout()`**：使用者選擇方案後，產生訂單並加密送往藍新金流。

**`handle_payment_notify()`**：藍新金流付款完成後，會打一個通知到這個端點，系統驗證後自動升級使用者方案。

**`handle_payment_return()`**：付款完成後瀏覽器跳轉回來的頁面，顯示付款結果。

### 2.16 代理 (Proxy) 功能

**`handle_proxy()`**：前端要從證交所、Yahoo Finance 等外部網站抓資料時，因為瀏覽器的跨域限制（CORS），不能直接抓。所以前端會把 URL 傳給後端，由後端去抓再轉傳回來。

- 有白名單限制，只允許存取特定主機
- 有伺服器端快取，避免頻繁打外部 API
- 對 TWSE 使用寬鬆的 SSL 設定（因為 TWSE 的 SSL 憑證有問題）
- 支援 Cookie 處理（應對 TWSE 的 WAF 防護）

### 2.17 伺服器啟動

檔案最底部是啟動流程：

1. 初始化資料庫
2. 啟動每日健檢排程（背景執行緒）
3. 啟動台指期報價輪詢（背景執行緒）
4. 啟動法人歷史資料回填（背景執行緒）
5. 啟動 HTTP 伺服器，監聽指定埠號
6. 初次啟動時執行一次健檢

## 三、前端邏輯 (js/app.js)

這是整個前端的核心邏輯，約 9,096 行。所有的資料抓取、計算、畫面渲染都在這裡。

### 3.1 設定與工具函式

**常數設定：**
- `TWSE` / `TPEX_OLD` / `TPEX_NEW`：證交所和櫃買中心的 API 網址
- `CACHE_MS`：前端快取有效期（10 分鐘）
- `MAX_CONCURRENT`：最多同時發出幾個 API 請求（6 個）

**數字格式化工具：**
- `parseNum(s)`：把字串轉成數字（處理逗號、加號、特殊符號）
- `fmtNum(n, dec)`：格式化數字加逗號
- `fmtBig(n)`：大數字格式化（超過 1 億顯示「X 億」，超過 1 萬顯示「X 萬」）
- `fmtShares(n)`：股數格式化（超過 1000 股顯示「X 張」）
- `limitTag(pct)` / `limitPrice(price, pct)`：漲停/跌停標籤

**日期工具：**
- `rocToISO(roc)`：民國日期轉西元（例如 115/03/20 → 2026-03-20）
- `dateStr(daysAgo)`：取得幾天前的日期字串
- `toTpexDate(yyyymmdd)`：西元日期轉櫃買中心格式（民國年/月/日）

**警示/處置股標記：**
- `loadWarningStocks()`：從 TWSE 載入注意股和處置股名單
- `warningTag(code)`：在股票旁邊顯示「警示」或「處置」標籤

### 3.2 iOS PWA 與捲動控制

- 防止 iOS PWA 模式下整個頁面被拖動彈跳
- 防止在 K 線圖上滾輪時整頁跟著捲動
- 防止桌面版水平捲動偏移

### 3.3 快取與網路請求

**`apiFetch(url)`**：核心的 API 請求函式

- 先查快取，有的話直接用
- 沒有就排入佇列，控制同時最多 6 個請求
- 所有外部 API 請求自動透過 `/api/proxy` 代理
- 每個請求有 12 秒逾時
- 失敗會自動重試一次（等 1.5 秒後）

### 3.4 API 呼叫定義

**`API_TWSE`** — 證交所 API：
- `instSummary(d)`：三大法人買賣超總表
- `instStocks(d)`：法人個股買賣超明細
- `allStocks(d)`：所有上市股票當日收盤行情
- `stockMonth(code, d)`：個股月K線資料
- `dayTrade(d)`：當沖統計與排行
- `sectorIndex(d)`：產業指數 5 分鐘資料
- `pePbYield(d)`：本益比、股價淨值比、殖利率
- `marginTrade(d)` / `marginSummary(d)`：融資融券

**`API_TPEX`** — 櫃買中心 API：
- `allStocks(d)`：所有上櫃股票當日行情
- `instStocks(d)`：法人個股買賣超
- `stockMonth(code, d)`：個股月K線

### 3.5 交易日偵測

**`findTradingDate()`**：自動找到最近一個有交易的日期

1. 先試 OpenAPI FMTQIK（最可靠，有日期欄位）
2. 再試 OpenAPI STOCK_DAY_ALL
3. 最後用傳統 BFI82U API 逐日嘗試（最多回查 7 天）

### 3.6 股票資料庫

**`buildStockDB()` / `searchStocks(query)`**

從載入的上市 + 上櫃股票資料建立一個本地搜尋資料庫。使用者輸入代號或名稱時，用這個資料庫做即時搜尋（最多 15 筆結果，代號完全符合的排最前面）。

### 3.7 技術分析指標 (TA 物件)

**`TA`** 物件包含所有技術分析的計算方法：

- **`sma(arr, p)`** — 簡單移動平均線（例如 MA5、MA10、MA20）
- **`ema(arr, p)`** — 指數移動平均線
- **`rsi(closes, p)`** — 相對強弱指標（RSI），預設 14 天。低於 30 是超賣，高於 70 是超買
- **`macd(closes)`** — MACD 指標，計算 DIF（12日EMA - 26日EMA）、Signal（DIF的9日EMA）、柱狀圖
- **`kd(H, L, C, p)`** — KD 隨機指標，預設 9 天。K 值和 D 值交叉是買賣訊號
- **`boll(closes, p, m)`** — 布林通道，預設 20 日均線 +/- 2 倍標準差

### 3.8 技術訊號偵測

**`detectSignals(C, H, L, V)`**：自動掃描最新的技術訊號

會偵測以下訊號（每個標記為看多 bullish、看空 bearish 或中性 neutral）：

- MA5/MA20 黃金交叉 / 死亡交叉
- MA10/MA20 黃金交叉 / 死亡交叉
- MACD 多方交叉 / 空方交叉
- KD 黃金交叉（低檔更強）/ 死亡交叉（高檔更強）
- RSI 超賣（< 30）/ RSI 超買（> 70）
- 突破布林上軌 / 跌破布林下軌
- 多頭排列（站穩所有均線）/ 空頭排列
- 爆量上漲 / 爆量下跌（量為均量 2 倍以上）

### 3.9 AI 評分系統

**`aiScore(C, H, L, V, inst)`**：簡易 AI 評分（滿分 100）

根據 6 個維度給分：
- RSI（最高 18 分）
- MACD（最高 20 分）
- KD（最高 20 分）
- 均線位置（最高 18 分）
- 量能（最高 10 分）
- 法人動向（最高 10 分）

**`scoreLabel(s)`**：根據分數給出評語（強力推薦 / 推薦買進 / 中性觀望 / 建議減碼 / 建議避開）

### 3.10 AI 深度分析報告

**`generateDeepAnalysis(...)`**：產生一份完整的個股分析報告 HTML

- 計算 7 個維度的分數（趨勢動能、RSI、MACD、KD、量能、法人動向、布林通道），每項 1~10 分
- 用加權平均算出綜合評分（趨勢和法人權重最高）
- 自動判定「強力看多 / 偏多操作 / 中性觀望 / 偏空保守 / 強力看空」
- 列出壓力區間和支撐區間
- 列出利多因素和利空因素
- 撰寫技術面總結文字
- 提供操作策略建議

### 3.11 關注清單

- `wlGet()` / `wlSave(list)`：從瀏覽器 localStorage 讀取/儲存關注清單
- `recentGet()` / `recentAdd(code)`：最近瀏覽的 10 檔股票紀錄

### 3.12 即時報價 (MIS)

**`fetchMisBatch(codes)`**：批次取得即時報價

1. 先嘗試 Fugle API（最多 30 檔）
2. Fugle 沒拿到的，再用 TWSE MIS（即時盤中資料）
3. MIS 每次最多查 20 檔，超過會分批，每批間隔 2 秒

**`fetchWatchlistMissing(codes)`**：關注清單中沒拿到即時報價的股票，用 Yahoo Finance 補足。

### 3.13 導航系統

**`switchTab(tabName, pushHistory, restoreScroll)`**：切換頁面分頁

- 支援瀏覽器上一頁/下一頁（用 History API）

### 3.14 自動完成搜尋

### 3.15 K 線圖系統

**`initCharts()`**：初始化四張圖表（使用 LightweightCharts 函式庫）

1. **主圖** — K 線（蠟燭圖）+ 成交量柱狀圖 + MA5/MA10/MA20 均線 + 布林通道
2. **RSI 副圖** — RSI 指標線
3. **KD 副圖** — K 值和 D 值兩條線
4. **MACD 副圖** — DIF 線 + Signal 線 + 柱狀圖

**`toggleChartSeries(key)`**：可以開關均線和布林通道的顯示。

**`toggleChartFullscreen()`**：全螢幕看 K 線圖（手機上會自動橫向旋轉）。

### 3.16 總覽頁面

**`renderOverview()`**：渲染市場總覽

### 3.18 個股分析

**`analyzeStock(code)`** — 個股分析主流程：

1. 判斷股票是上市、上櫃還是興櫃
2. 抓取 8 個月的歷史 K 線資料（優先用 TWSE/TPEX，備援用 Yahoo）
3. 同時抓取即時報價
4. 去除重複日期並排序
5. 解析 OHLCV（開高低收量）
6. 在 K 線圖上畫出蠟燭圖、均線、布林通道
7. 計算並畫出 RSI、KD、MACD 副圖
8. 偵測技術訊號
9. 計算 AI 評分
10. 顯示法人買賣超
11. 產生 AI 深度分析報告
12. 產生籌碼速覽（趨勢、量能、訊號摘要）
13. 背景載入：營收、季度財報、基本面、融資融券、股利、新聞、事件標記
14. 啟動即時更新（每 10 秒刷新報價）

**`fetchStockRevenue(code)`**：載入最新月營收 + 營收歷史趨勢圖（近 26 個月）。

**`fetchStockQuarterly(code)`**：載入最新季度財報（EPS、營收、淨利率）+ EPS 趨勢圖（近 3 年）。

**`fetchStockFundamentals(code)`**：載入本益比（PE）、股價淨值比（PB）、殖利率。

**`fetchStockMargin(code)`**：載入融資融券資料（餘額、增減、使用率、券資比）。

**`fetchStockDividend(code)`**：載入近 5 年股利政策（現金股利 + 股票股利）。

**`fetchStockNews(code)`**：載入個股相關新聞，重要新聞（法說、併購、漲跌停等）會加上標記。

**`fetchStockEvents(code, dates)`**：在 K 線圖上標記法說會和除息日。

**`renderChipSummary(...)`**：籌碼速覽卡片（AI 評分、趨勢判定、量能狀態、技術訊號、法人長條圖）。

## 五、樣式 (css/style.css)

約 4,413 行的 CSS，採用深色科技風格主題，全手寫（無 CSS 框架）。

### 5.1 CSS 變數 (根目錄 :root)

定義全站顏色系統：

- `--bg`：主背景色（深藍黑 #060b18）
- `--card`：卡片背景（半透明深藍）
- `--border`：邊框（青色半透明）
- `--text` / `--text2`：主/次文字色

- `--cyan` / `--purple` / `--yellow` 等：強調色
- 各種 glow 光暈效果

**注意**：台灣股市慣例是紅色代表上漲、綠色代表下跌，與歐美相反。CSS 裡的 `.up` 類別是紅色，`.down` 類別是綠色。

## 九、重要概念補充

### 9.1 資料流程（以「看個股分析」為例）

1. 使用者在搜尋框輸入「2330」按 Enter
2. `analyzeStock('2330')` 被呼叫
3. 判斷 2330 是上市股票（在 gStockDB 裡查到）
4. 平行發出請求：
   - 歷史 K 線：打 TWSE 的 STOCK_DAY API（8 個月份同時抓）
   - 即時報價：打 Fugle API
5. TWSE API 請求經過 `/api/proxy` → `server.py` 的 `handle_proxy()` → 再打 TWSE
6. 前端收到資料後：解析 OHLCV → 計算技術指標 → 畫 K 線圖 → 偵測訊號 → AI 評分
7. 背景繼續載入：營收、財報、融資融券、新聞等

|------|------|-----|------|

### 9.3 錯誤處理策略

- 外部 API 失敗時會自動重試一次
- TWSE 被擋時會改用 FinMind 或 Yahoo Finance 備援
- 個股分析如果 TWSE 沒資料會嘗試 TPEX，再嘗試 Yahoo
- 所有 API 端點都有 try/catch，不會因為單一錯誤導致整個伺服器崩潰
- 網路斷線時 Service Worker 會回傳離線提示
